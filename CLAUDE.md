## Architecture

### Layer Overview
```
UI (features/) ‚Üí State (Riverpod providers) ‚Üí Services ‚Üí Data (SQLite / Firebase / SharedPreferences)
```

### Key Providers (`lib/core/providers/auth_provider.dart`)
- `activeChildProvider` ‚Äî selected child profile
- `timerProvider` ‚Äî countdown notifier
- `settingsServiceProvider` ‚Äî SharedPreferences wrapper
- `questionServiceProvider` ‚Äî routes to question source

### Question Source Strategy (`lib/core/services/question_service.dart`)
`QuestionService` delegates to one of four `QuestionSourceStrategy` implementations:
- `FixedBankSource` ‚Äî SQLite (pre-loaded from `assets/questions/questions.json`)
- `AIGeneratedSource` ‚Äî Firebase Cloud Function ‚Üí Claude API
- `PhotoSource` ‚Äî image upload ‚Üí Claude vision ‚Üí question
- Mixed mode

### Core User Flow
1. Child selects avatar (`child_select_screen.dart`)
2. Enters YouTube URL on home screen
3. **Full-screen quiz overlay appears** (cannot be dismissed) ‚Äî child must answer correctly
4. Timer starts (default 30 min, configurable by parent)
5. Timer expires ‚Üí overlay reappears; cycle repeats

### Platform Differences
- Mobile: native YouTube player
- Web: iframe implementation (`web_player.dart`)
- Stub files in `lib/core/services/stub/` handle conditional compilation for Web/Desktop

### Parent Mode
PIN-protected (4‚Äì6 digits). Accessible from any screen. Settings include: session duration, question count to unlock, subject/grade/unit selection, question source, child photo upload.

### Debug Override
`lib/core/config/app_config.dart` contains `debugDurationSeconds = 30` ‚Äî sets session timer to 30 seconds for testing. Set to `null` for production behavior.

### Firebase Cloud Functions (`lib/functions/src/index.ts`)
Two callable functions:
- `generateQuestion` ‚Äî AI question generation with Firestore caching
- `generateFromPhoto` ‚Äî vision-based question generation from uploaded curriculum photos
